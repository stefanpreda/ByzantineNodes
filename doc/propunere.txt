-------------------------INIT---------------------------------
(DONE)BASE STATION TRIMITE MESAJ DE ACTIVARE NODURILOR IN CARE SE DA LISTA DE NODURI + RANG PENTRU FIECARE(fiecare nod primeste un map ID:RANK cu toate nodurile)
(DONE)DUPA MESAJUL DE ACTIVARE SE TRIGGER-UIESTE ALEGEREA INITIALA A LIDERULUI

------------------------LEADER SELECTION----------------------
SE TRIGGERUIESTE IN 4 CAZURI:
	(DONE)INITIAL DE BASE STATION
	VECHIUL LIDER TRIMITE REQUEST SI NU SE INSCRIE IN PROCESUL DE ALEGERE (SAU ESTE IGNORAT DACA ESTE BYZANTIN SI SE INSCRIE IAR)
	ESTE TRIGGERUIT DE BASE STATION DACA NU MAI PRIMESTE MESAJE (CAZ IN CARE LIDERUL CURENT ESTE BYZANTIN SI NU TRIMITE MAI DEPARTE LA BASE STATION, SAU A MURIT), LIDERUL CURENT VA FI IGNORAT LA SELECTIE
	PRIN MESAJE DE LEADER SELECTION PROPOSAL DACA DEPASESC UN ANUMIT THRESHOLD (PESTE 50% NODURI PROPUN ALEGEERA ALTUI LIDER)


(DONE)SE FACE FLOOD CU MESAJE DE CANDIDARE LA LEADERSHIP
(DONE)FIRECARE NOD DETERMINA DIN MESAJE SI RANGURILE MENTINUTE DE EL CARE ESTE NOUL LIDER
(DONE)SE ALEGE LIDER NODUL CU RANG CEL MAI MARE
	(DONE)CASE 1: NOD BYZANTIN TRIMITE VALOARE MAI MARE DECAT CEA SALVATA INTERN --->> ESTE ELIMINAT DIN PROCESUL DE SELECTIE
	(DONE)CASE 2: UN NOD TRIMITE IN MESAJ O VALOARE MAI MICA DECAT CEA SALVATA INTERN --->> SE MODIFICA INTERN
				(DONE)1. NODUL CURENT A AVUT O DEFECTIUNE SI NU A MODIFICAT CORECT VALOAREA SI SE REAJUSTEAZA
				(DONE)2. NODUL SENDER ESTE BYZANTIN SI DACA II MICSOREZ VALOAREA ATUNCI NU VA FI ALES CA LIDER (COMPORTAMENT DORIT)

(DONE)SE FACE FLOOD CU LIDERUL SELECTAT
(DONE)FIECARE NOD SELECTEAZA LIDERUL CU CELE MAI MULTE MESAJE SI MENTINE IN MEMORIE IDENTITATEA LUI 

(DONE)DUPA CE UN LIDER ISI PIERDE POZITIA SI UNUL NOU ESTE ALES FIECARE NOD II SCADE RANGUL CU INTREAGA VALOARE (E.G rang=5 -> rang=0 -> rang=-5 etc) PENTRU A CICLA PRIN LIDERI

DACA UN NOD BYZANTIN SELECTEAZA ALT LIDER NU INFLUENTEAZA FUNCTIONALITATEA RETELEI, VA TRIMITE MESAJE CARE VOR FI DISCARDED
DACA UN NOD BYZANTIN NU ISI MODIFICA INTERN RANGUL FOSTULUI LIDER ATUNCI
	1. IL VA IGNORA IN PROCESUL DE DECIZIE
	2. IL VA CONSIDERA LIDER IN PROCESUL DE DECIZIE
	IN AMBELE CAZURI NU INFLUENTEAZA FUNCTIONALITATEA RETELEI ATATA TIMP CAT NUMARUL DE NODURI BYZANTINE ESTE SUFICIENT DE MIC

(DONE)SE ACCEPTA UN SINGUR MESAJ DE LA FIECARE NOD IN PROCESUL DE DECIZIE (IN CAZ CA NODURILE BYZANTINE VOR TRIMITE MESAJE INCONTINUU)

-------------------------ROUTING--------------------------
TOATE PROBLEMELE DE ROUTING (RUTARE GRESITA, NOD OFF, NU RUTEAZA) SE POT REZOLVA PRIN FOLOSIREA UNUI ALGORITM (E.G protocolul SMT) DE RUTARE IN RETELE NESIGURE SI VOR FI IGNORATE
IMPLEMENTAREA SE AXEAZA PE ALGORITMUL DE LA NIVEL APLICATIE

NODUL LIDER TRANSMITE LA BASE STATION
BASE STATION TRANSMITE UN MESAJ DE ACK LA TOATE NODURILE IN CARE SPECIFICA VALOAREA PRIMITA
DACA UN NOD PRIMESTE ALTA VALOARE DECAT CEA DE LA BASE STATION DECAT CEA DE CARE STIA EL CA S-A CONVENIT ATUNCI FACE FLOOD CU UN MESAJ DE LEADER SELECTION PROPOSAL
DACA SE ALEGE UN NOU LIDER IN URMA PROPUNERII ANTERIOARE ATUNCI SE TRIMITE MESAJ LA BASE STATION SA IGNORE VALOAREA ANTERIOARA SI SE REFACE PROCESUL

(DONE)SE ACCEPTA UN SINGUR MESAJ DE LA FIECARE NOD IN PROCESUL DE DECIZIE (IN CAZ CA NODURILE BYZANTINE VOR TRIMITE MESAJE INCONTINUU)

BASE STATION RECEIVES MESSAGE FROM ANOTHER NODE, SENDS ACK MESSAGE, NODES WITH DIFFERENT LEADER ANNOUNCE BASE STATION
	-NUMBER ANNOUNCED > THRESHOLD => DROP MESSAGE
	-NUMBER ANNOUNCED < THRESHOLD => SEND MESSAGE TO ADJUST LEADER
